# geo-calcu
根据经纬度计算距离的三种方式

| 单线程-100次   | mysql    | redis (14M) | memory   |
|------------|----------|-------------|----------|
| 内存占用       | 50M~160M | 50M~270M    | 84M~340M |
| max ms     | 863      | 132         | 606      |
| min ms     | 347      | 56          | 19       |
| avg ms     | 376      | 61          | 40       |
| fetch 500  | -        | 17s~27s     | -        |
| fetch 1000 | -        | 19s~32s     | -        |

## 个人建议

**1. mysql计算方案，单体架构，数据量很少，不想引入第三方中间件。**

**2. redis计算方案，距离计算方案没有定制化要求。**

**3. 内存计算方案，数据量巨大，位置不固定，对实时性要求高，有定制化需求。**
